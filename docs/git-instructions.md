# Git ワークフローガイドライン

## コミットメッセージ規約

### 基本原則
- **客観性の維持**: 技術的事実のみを記述し、品質評価を含めない
- **専門性の確保**: 第三者が見た際の信頼性を重視した記述
- **機能中心**: 実装内容・解決した問題・技術仕様に限定

### 禁止表現

#### 評価的表現の禁止
❌ **使用禁止**: 素晴らしい、完璧、最高、美しい、エレガント、世界最高峰、究極
❌ **主観的評価**: 改善、向上、最適化 → 具体的変更内容を記述
❌ **感情的表現**: 感動、驚き、満足、達成感等の感情表現

#### 適切な表現
✅ **機能記述**: 実装、追加、修正、変更、対応
✅ **技術仕様**: アルゴリズム、データ構造、パフォーマンス指標
✅ **問題解決**: バグ修正、制限事項の解決、仕様対応

### Conventional Commits準拠

#### タイプ分類
- `feat`: 新機能の追加
- `fix`: バグ修正
- `refactor`: 機能変更を伴わないコード改善
- `perf`: パフォーマンス改善
- `style`: コードスタイル調整（フォーマット、セミコロン等）
- `docs`: ドキュメント更新
- `test`: テスト追加・修正
- `chore`: ビルドプロセス・補助ツール変更

#### メッセージ構造
```
<type>(<scope>): <description>

<body>

<footer>
```

### コミットメッセージ例

#### ❌ 不適切な例
```
feat: 素晴らしいタロットカードUIの実装

世界最高峰のカード表示システムを構築しました。
完璧なレスポンシブ対応で美しいアニメーションを実現。
```

#### ✅ 適切な例
```
feat: タロットカード横スクロール表示システム

- 160px/208pxレスポンシブカードサイズ対応
- スクロール位置連動のナビゲーション状態管理
- モバイル画面幅制御によるレイアウト破綻対策
- CSS scrollbar-hide実装
```

### ブランチ戦略

#### ブランチ命名規則
- `feature/機能名`: 新機能開発
- `fix/問題内容`: バグ修正
- `refactor/対象`: リファクタリング
- `docs/更新内容`: ドキュメント更新

#### 作業フロー
1. **feature/ui-improvements**から新ブランチ作成
2. 機能実装・テスト実行
3. セルフレビュー実施
4. プルリクエスト作成
5. コードレビュー対応
6. マージ実行

### プルリクエスト規約

#### タイトル形式
```
[Type] 機能/修正内容の簡潔な説明
```

#### 説明テンプレート
```markdown
## 変更内容
- 実装した機能の具体的説明
- 修正した問題の詳細
- 技術的な実装アプローチ

## 技術仕様
- 使用技術・ライブラリ
- パフォーマンス影響
- 破壊的変更の有無

## テスト計画
- 検証項目のチェックリスト
- 動作確認環境
- 回帰テスト結果

## レビュー観点
- コード品質チェックポイント
- セキュリティ考慮事項
- パフォーマンス影響評価
```

### コードレビュー指針

#### レビュー観点
1. **機能性**: 仕様要件の充足度
2. **保守性**: コードの可読性・拡張性
3. **性能**: パフォーマンス・メモリ使用量
4. **安全性**: セキュリティ・エラーハンドリング
5. **規約**: スタイリングガイド・命名規則の遵守

#### フィードバック方針
- **建設的批判**: 改善提案を具体的に提示
- **客観的評価**: 技術的根拠に基づく指摘
- **学習機会**: ベストプラクティスの共有

### リリース管理

#### semantic-release設定
- **patch**: バグ修正・小改善
- **minor**: 新機能追加・非破壊的変更
- **major**: 破壊的変更・大規模リファクタリング

#### リリースノート自動生成
- Conventional Commitsに基づく自動分類
- 機能追加・バグ修正・破壊的変更の明確な区分
- 技術仕様中心の記述

### 品質保証プロセス

#### 継続的改善
- **メトリクス監視**: コード品質・パフォーマンス指標
- **定期レビュー**: 実装パターン・設計原則の見直し
- **技術負債管理**: 改善優先度の定期評価

#### 文書化要件
- **技術仕様書**: アーキテクチャ・API仕様
- **運用手順書**: デプロイ・監視・障害対応
- **変更履歴**: 意思決定記録・技術選択根拠
