## 📝 仕様書の作成ガイドライン

新しい仕様書を追加する際は、以下のガイドラインに従ってください：

### 配置場所

- **フック**: `hooks/[フック名].md`
- **コンポーネント**: `components/[コンポーネント名].md`
- **API**: `api/[API名].md`
- **機能**: `features/[機能名].md`

### 命名規則

- ファイル名はPascalCase（例: `useFortune.md`, `MessageDialog.md`）
- 対象コードと同じ名前を使用

### 必須セクション

仕様書には以下のセクションを含めてください：

1. **概要**: 作成日、対象バージョン、ファイルパス
2. **機能概要**: 何をするものか、なぜ必要か
3. **インターフェース**: 使用方法、引数、戻り値
4. **動作仕様**: 正常系フロー、エラーハンドリング
5. **データ構造**: 使用する型定義
6. **技術詳細**: 依存関係、パフォーマンス特性
7. **実装上の注意点**: セキュリティ、パフォーマンス、保守性
8. **テストケース**: 正常系、異常系、エッジケース

### コードへの参照追加

対象のコードファイルに、仕様書へのリンクを追加してください：

```typescript
/**
 * [説明]
 *
 * @see 仕様書: docs/specs/[カテゴリ]/[ファイル名].md
 */
```
