# Git Push Automation

## 現在の状態確認
- Git status: !`git status`
- 未ステージ差分: !`git diff`
- ステージ済み差分: !`git diff --staged`
- 現在のブランチ: !`git branch --show-current`
- 最新コミット: !`git log --oneline -5`

## プロジェクト設定の確認
- Gitフロー設定: @.gitflow (存在する場合)
- プロジェクト設定: @.git/config
- コミット規約: @CONTRIBUTING.md (存在する場合)
- README: @README.md (gitフロー情報があるか確認)

## タスク
1. **差分分析**: 変更内容を詳細に分析し、何が変更されたかを理解する
2. **gitフロー確認**: プロジェクトのgitフロー設定やコミット規約を確認する
3. **コミットメッセージ生成**: 
   - プロジェクトの規約に従った形式
   - 変更内容を適切に説明
   - 必要に応じてConventional Commits形式を使用
4. **ステージング**: 適切なファイルをステージング
5. **コミット実行**: 生成したメッセージでコミット
6. **プッシュ実行**: 現在のブランチにプッシュ

## 実行手順
- 変更ファイルを確認し、必要に応じて選択的にステージング
- プロジェクトのコミット規約に従ったメッセージを生成
- コミットとプッシュを実行
- 実行結果を確認し、問題があれば報告

注意: プッシュ前に必ず変更内容を確認し、適切性を検証してください。
